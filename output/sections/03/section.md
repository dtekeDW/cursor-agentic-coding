## Lernziel
- Teilnehmer können für eine Änderung einen kleinen, überprüfbaren Plan erzeugen.

## Kernbotschaft
- Plan-First ist ein Qualitäts-Gate: erst Optionen, dann Umsetzung.

## Live-Demo-Setup
- Kleines Demo-Repo-Szenario mit einer klar abgegrenzten Änderung.

## Demo-Skript
1. Ask Mode: 2-3 Umsetzungsoptionen erzeugen.
2. Risiken vergleichen, Option auswählen.
3. Plan in Phasen (inkl. Tests) bestätigen.

## Prompt/Command-Beispiele
```text
Use Ask mode first.
Do not edit code yet.
Give me 2-3 implementation options with risks, affected files, and recommendation.
```

```text
Now switch to Plan mode and break the chosen option into minimal phases.
```
